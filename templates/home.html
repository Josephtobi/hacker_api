Hello, I wanted to walk you through how i approached the problem.... Enjoy
Beautiful project that came with its own share of fun and learnings.
The project idea is to consume an api from hacker news and save the data (news, comments and sub_comments) into the db every 5seconds
  • this wasnt easy as celery didnt work on replit, (setting up rabbitmq was the issue here). so here is what i did
  • i created a task.py file that consumed the api and tied it to a view that runs it everytime its visited.
  • then scheduled uptimebot to visit the page every 5mins
    • The next task was to create API endpoints
  • This was easily done with DRF
  • The filter on the get request feature was tied to another view incase you are wondering why i have two get requests url
    • Search feature on news title was a breeze but filter was not implemented because all types displayed on news page are news only
    And that sums it up for me, i hope to hear from you soon
